article-formatted-body article-formatted-body_version-2"><div xmlns="http://www.w3.org/1999/xhtml"><p>Microsoft <a href="https://support.microsoft.com/en-us/office/classic-outlook-error-when-sending-from-outlook-com-you-do-not-have-permission-to-send-on-behalf-of-the-specified-user-0x80070005-0x0004dc-0x000524-49f5afe9-11ee-4ce1-9531-296670171509" rel="noopener noreferrer nofollow" target="_blank">подтвердила</a> проблему в классическом Outlook, которая блокирует отправку и ответ на электронные письма с сообщением «У вас нет разрешения на отправку от имени указанного пользователя 0x80070005-0x0004dc-0x000524». Это происходит в веб-версии, когда пользователи пытаются отправить письмо с Outlook.com. Компания заявляет, что ошибка связана с недоставкой отчёта (NDR) и возникает гораздо чаще, когда учётная запись Outlook.com находится в профиле Outlook с другой учётной записью Exchange.</p><figure class="full-width "><img src="https://habrastorage.org/r/w1560/getpro/habr/upload_files/e8d/805/cd1/e8d805cd1721a18cc10d0f5990b1aee3.JPG" width="909" height="612" sizes="(max-width: 780px) 100vw, 50vw" srcset="https://habrastorage.org/r/w780/getpro/habr/upload_files/e8d/805/cd1/e8d805cd1721a18cc10d0f5990b1aee3.JPG 780w,
       https://habrastorage.org/r/w1560/getpro/habr/upload_files/e8d/805/cd1/e8d805cd1721a18cc10d0f5990b1aee3.JPG 781w" loading="lazy" decode="async"></figure><p>В связи с этим компания предложила несколько обходных решений, пока проводится расследование. В Microsoft указывают: «Эта проблема возникает, если учётная запись Outlook.com, с которой вы отправляете письма, находится в профиле Outlook с учётной записью Microsoft 365, и у этого аккаунта есть почтовый контакт Exchange Online с тем же SMTP-адресом».&nbsp;</p><p><strong>Решение № 1:</strong></p><ul><li><p>создать новое электронное письмо;</p></li><li><p>выбрать раскрывающийся список «От», затем выбрать «Другой адрес электронной почты…» и «От…»;</p></li><li><p>в разделе «Адресная книга:» выбрать «Глобальный список адресов» для другой учётной записи Microsoft 365 в профиле, выбрать адрес электронной почты учётной записи, которая получает уведомление о недоставке (NDR), затем два раза нажать «ОК».</p></li></ul><p><strong>Решение № 2 — удаление адресной книги учётной записи M365, чтобы Outlook не проверял её при отправке электронных писем:</strong></p><ul><li><p>выбрать папку «Входящие» учётной записи Outlook.com;</p></li><li><p>на ленте Outlook выбрать «Адресная книга» в столбце «Найти»;</p></li><li><p>перейти в «Инструменты» &gt; «Параметры» &gt; «Настраиваемые»;</p></li><li><p>удалить элементы с «Глобальным списком адресов» и нажать «ОК» после завершения.&nbsp;</p></li></ul><p><strong>Решение №3 — скрыть контакт Outlook.com из глобального адресного списка (GAL) учётной записи Microsoft 365:</strong></p><ul><li><p>перейти в Центр администрирования Microsoft 365 и войти в систему с учётной записью администратора Microsoft 365;</p></li><li><p>на главной странице портала администрирования найти «Центры администрирования», выбрать «Показать все», а затем выбрать Exchange;</p></li><li><p>на главной странице портала администрирования Exchange выбрать «Контакты» в навигации;</p></li><li><p>найти контакт для адреса электронной почты, на который приходит уведомление о недоставке (NDR), выбрать его;</p></li><li><p>в диалоговом окне «Почтовый контакт» для этого контакта найти «Управление скрытием из GAL»;</p></li><li><p>выбрать «Управление скрытием из GAL», активировать переключатель для параметра «Скрывать из глобального адресного списка», нажать «Сохранить»;</p></li><li><p>чтобы Outlook отобразил это изменение, нужно вернуться в свой профиль Outlook, выбрать папку «Входящие» учётной записи Microsoft 365. На ленте выбрать «Отправить/получить» &gt; «Группы отправки/получения» &gt; «Загрузить адресную книгу», затем нажать «ОК»;</p></li><li><p>чтобы внести изменения, нужно выбрать «Главная» на ленте, затем «Адресная книга». В разделе «Адресная книга» потребуется найти учётную запись Microsoft 365. Затем нужно выбрать «Автономная адресная книга» для своей учётной записи Microsoft 365. Учётная запись Outlook.com больше не должна отображаться там.</p></li></ul><p>В компании отметили, что, когда найдётся решение проблемы, пользователь сможет повторно включить этот контакт в глобальном списке адресов, выполнив те же действия, что и выше.</p><p>Помимо этого, Microsoft также предлагает использовать Outlook.com из веб-браузера или создать новый профиль в классической версии, но только с той учётной записью, которая не вызывает ошибку «Недоставлено сообщение».</p><p>Наконец, компания предлагает пользователям использовать новый Outlook для Windows, чтобы отправлять электронные письма с проблемной учётной записи.&nbsp;</p><p>Ранее Microsoft <a href="https://habr.com/ru/news/1015276/" rel="noopener noreferrer nofollow">исправила</a> ошибку классического Outlook, которая не позволяла синхронизировать контент учётных записей с Gmail и Yahoo. Пользователи впервые столкнулись с проблемами 26 февраля. При добавлении учётных записей Gmail и Yahoo в почтовый клиент они не могли синхронизировать свой контент. Это происходило даже после сброса пароля, и при этом не требовалось повторного входа в систему.  </p></div></div></div></div><div data-v-aad06d04="" aria-hidden="true" class="pswp" role="dialog"
