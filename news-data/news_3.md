Anthropic опинилася в центрі обговорень після того, як в одній із версій npm-пакета Claude Code випадково був опублікований файл cli.js.map. Саме через нього дослідники змогли отримати доступ до вихідного коду інструмента, і в мережу потрапило понад 500 тисяч рядків TypeScript-коду. За наявною інформацією, йдеться приблизно про 1 900 файлів.

Важливо, що це не був злам у класичному розумінні. Проблема виникла через помилку в процесі збірки пакета: у фінальний реліз потрапив source map, який зазвичай використовується для дебагу та спрощення роботи розробників. Такі файли можуть містити не лише посилання на вихідники, а й сам код, структуру проєкту, назви функцій і внутрішню архітектуру застосунку.

З практичної точки зору це неприємний інцидент для Anthropic, оскільки Claude Code — це не просто CLI-утиліта, а важлива частина їхньої developer-екосистеми. Подібні витоки дозволяють значно глибше зрозуміти внутрішню реалізацію продукту, архітектурні рішення та підходи компанії до побудови AI-інструментів для розробки.

Для індустрії це ще одне нагадування, що помилки в пайплайнах збірки та публікації можуть бути не менш небезпечними, ніж класичні security-вразливості. Іноді достатньо одного зайвого артефакту в npm-пакеті, щоб назовні вийшло те, що взагалі не планувалося публікувати.
